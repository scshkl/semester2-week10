# Profiling

The output generated by Gprof is rather plain and uninteresting. Investigate
nicer ways of visualizing the call graph that it generates.

For example, you could experiment with using [Graphviz][gviz] to display
the call graph.

The Python tool [gprof2dot][g2d] can be used to process the output from
`gprof` and generate the 'DOT file' needed by Graphviz. You could download
the `gprof2dot.py` script and then generate a call graph like this:

    gprof prog gmon.out | python3 gprof2dot.py | dot -Tpng -o calls.png

This should work on SoCS Linux machines.


[gviz]: https://graphviz.org/
[g2d]: https://github.com/jrfonseca/gprof2dot
